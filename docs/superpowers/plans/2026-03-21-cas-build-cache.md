# CAS-Backed Build Cache Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate the 15-30 minute iroh-relay build on e2-micro by using Nix binary caching — build once on a workstation, serve from the relay VM.

**Architecture:** The existing `flake.nix` already builds `iroh-relay-x86_64-linux` via crane (musl static binary). A new one-time setup script installs multi-user Nix and `nix-serve` on the relay VM. The deploy script is rewritten to: (1) check if the store path exists on the VM, (2) build locally via `nix build` and push via `nix-store --export | gcloud ssh | nix-store --import`, (3) activate. All cargo build paths removed.

**Tech Stack:** Nix, nix-serve, gcloud CLI, bash

**Spec:** `docs/superpowers/specs/2026-03-21-cas-build-cache-design.md`

- [ ] **Step 1: Create the setup script**

Create `deploy/relay/nix-cache-setup.sh`:

```bash
#!/usr/bin/env bash
# nix-cache-setup.sh — One-time setup of Nix binary cache on relay VM.
#
# Usage: Run via gcloud compute ssh from the deploy machine:
#   gcloud compute ssh VM_NAME --zone=ZONE -- "bash -s" < deploy/relay/nix-cache-setup.sh
#
# What it does:
#   1. Installs multi-user Nix (with daemon) if not present
#   2. Generates binary cache signing keypair if not present
#   3. Installs nix-serve from nixpkgs
#   4. Creates systemd service for nix-serve on port 5000
#   5. Prints the public key for the deployer to commit

set -euo pipefail

CACHE_KEY_DIR="/etc/nix"
CACHE_KEY_PRIVATE="${CACHE_KEY_DIR}/cache-key.pem"
CACHE_KEY_PUBLIC="${CACHE_KEY_DIR}/cache-key.pub"
NIX_SERVE_PORT=5000

echo "=== Nix Binary Cache Setup ==="

# ── 1. Install Nix (multi-user) ──────────────────────────────────────────
if command -v nix &>/dev/null; then
    echo "[1/5] Nix already installed: $(nix --version)"
else
    echo "[1/5] Installing Nix (multi-user)..."
    sh <(curl -L https://nixos.org/nix/install) --daemon --yes
    echo "    Nix installed. Sourcing profile..."
    # shellcheck disable=SC1091
    . /etc/profile.d/nix.sh 2>/dev/null || . "$HOME/.nix-profile/etc/profile.d/nix.sh" 2>/dev/null || true
fi

# Ensure nix commands work in this session
export PATH="/nix/var/nix/profiles/default/bin:$PATH"

# ── 2. Generate signing keypair ───────────────────────────────────────────
if [ -f "$CACHE_KEY_PRIVATE" ]; then
    echo "[2/5] Signing key already exists at ${CACHE_KEY_PRIVATE}"
else
    echo "[2/5] Generating binary cache signing keypair..."
    sudo mkdir -p "$CACHE_KEY_DIR"
    nix-store --generate-binary-cache-signing-key "$CACHE_KEY_PRIVATE"
    # Extract public key (the .pub is not auto-generated by all versions)
    if [ ! -f "$CACHE_KEY_PUBLIC" ]; then
        # The private key file contains "cache-name:base64-secret-key"
        # The public key is derived by nix — for now, we'll generate it
        # when first signing. The deployer extracts it from narinfo responses.
        echo "    (Public key will be extracted after first signing)"
    fi
    sudo chmod 600 "$CACHE_KEY_PRIVATE"
    echo "    Signing key generated."
fi

# ── 3. Install nix-serve ──────────────────────────────────────────────────
if command -v nix-serve &>/dev/null || nix path-info nixpkgs#nix-serve &>/dev/null 2>&1; then
    echo "[3/5] nix-serve already available."
else
    echo "[3/5] Installing nix-serve..."
    nix profile install nixpkgs#nix-serve
fi

# ── 4. Create systemd service ────────────────────────────────────────────
echo "[4/5] Configuring nix-serve systemd service..."
sudo tee /etc/systemd/system/nix-serve.service > /dev/null << EOF
[Unit]
Description=Nix Binary Cache (nix-serve)
After=network-online.target nix-daemon.service
Wants=network-online.target

[Service]
Type=simple
ExecStart=$(command -v nix-serve || echo /nix/var/nix/profiles/default/bin/nix-serve) --port ${NIX_SERVE_PORT}
Restart=always
RestartSec=5s

# Run as root to read /nix/store (nix-serve needs store access)
User=root

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable nix-serve
sudo systemctl restart nix-serve

echo "    nix-serve running on port ${NIX_SERVE_PORT}"

# ── 5. Print public key ──────────────────────────────────────────────────
echo ""
echo "[5/5] Setup complete."
echo ""
echo "=== IMPORTANT ==="
echo "Add this firewall rule if not already present:"
echo "  gcloud compute firewall-rules create <vm>-nix-cache --allow=tcp:${NIX_SERVE_PORT} --target-tags=<vm>"
echo ""
if [ -f "$CACHE_KEY_PRIVATE" ]; then
    echo "Signing key: ${CACHE_KEY_PRIVATE}"
    echo "To extract the public key, run on this VM:"
    echo "  cat ${CACHE_KEY_PRIVATE} | sed 's/:.*/:/' "
    echo ""
    echo "Then commit the public key to deploy/relay/cache-key.pub"
fi
echo ""
echo "Binary cache URL: http://<vm-ip>:${NIX_SERVE_PORT}"
echo "Test: curl http://localhost:${NIX_SERVE_PORT}/nix-cache-info"
```

```bash
# ── Step 4: Build iroh-relay ──────────────────────────────────────
# Two strategies, in order of preference:
#   1. Use a pre-built binary (LOCAL_BINARY env var)
#   2. Build with Nix and push to VM
#      a. Check if store path already exists on VM (fast path)
#      b. Build locally with nix build, export/import via gcloud ssh

BINARY_PATH=""
BUILD_STRATEGY=""

if [ -n "$LOCAL_BINARY" ] && [ -f "$LOCAL_BINARY" ]; then
    echo "--- Step 4: Using pre-built binary: ${LOCAL_BINARY}"
    BINARY_PATH="$LOCAL_BINARY"
    BUILD_STRATEGY="prebuilt"

elif command -v nix &>/dev/null; then
    echo "--- Step 4: Building iroh-relay with Nix..."

    # Get the store path without building (from eval)
    STORE_PATH=$(nix build .#iroh-relay-x86_64-linux --print-out-paths --no-link 2>/dev/null || true)

    if [ -z "$STORE_PATH" ]; then
        echo "    Building .#iroh-relay-x86_64-linux (first time takes ~2 min)..."
        STORE_PATH=$(nix build .#iroh-relay-x86_64-linux --print-out-paths --no-link)
    fi

    echo "    Store path: ${STORE_PATH}"

    # Check if VM already has this store path
    VM_HAS_PATH=$(gcloud compute ssh "$VM_NAME" --zone="$GCP_ZONE" --command="
        nix path-info '$STORE_PATH' &>/dev/null 2>&1 && echo yes || echo no
    " 2>/dev/null || echo "no")

    if [ "$VM_HAS_PATH" = "yes" ]; then
        echo "    VM already has this build — skipping copy."
        BUILD_STRATEGY="nix-cached"
    else
        echo "    Pushing closure to VM (via gcloud ssh)..."
        nix-store --export $(nix-store -qR "$STORE_PATH") | \
            gcloud compute ssh "$VM_NAME" --zone="$GCP_ZONE" -- \
            "sudo nix-store --import" > /dev/null
        echo "    Closure pushed."

        # Sign the paths on the VM
        CACHE_KEY="/etc/nix/cache-key.pem"
        gcloud compute ssh "$VM_NAME" --zone="$GCP_ZONE" --command="
            if [ -f '$CACHE_KEY' ]; then
                sudo nix store sign --key-file '$CACHE_KEY' '$STORE_PATH'
                echo 'Store path signed.'
            else
                echo 'WARNING: No signing key found at $CACHE_KEY — skipping signing.'
            fi
        "
        BUILD_STRATEGY="nix-push"
    fi

    # Set BINARY_PATH to the binary inside the store path
    BINARY_PATH="${STORE_PATH}/bin/iroh-relay"

else
    echo "ERROR: Nix not found. Install Nix (https://nixos.org/download) or set LOCAL_BINARY."
    echo "  curl -L https://nixos.org/nix/install | sh"
    exit 1
fi
```

## Binary Cache

The relay VM runs a Nix binary cache on port 5000. Once set up, builds are
cached — subsequent deploys skip the build step entirely.

### First-Time Setup

Run the setup script on the relay VM:

```bash
gcloud compute ssh $VM_NAME --zone=$GCP_ZONE -- "bash -s" < deploy/relay/nix-cache-setup.sh
```

This installs Nix, `nix-serve`, and generates a signing keypair. Then add
the firewall rule (deploy.sh does this automatically on subsequent runs).

### How It Works

1. `deploy.sh` runs `nix build .#iroh-relay-x86_64-linux` locally (~2 min first time)
2. Checks if the VM already has this exact build (by store path hash)
3. If not, pushes the closure via `gcloud compute ssh` pipe
4. Activates the binary and restarts the service

The binary cache at `http://i.q8.fyi:5000` serves cached builds to any
Nix-enabled machine:

```nix
# In nix.conf
substituters = https://cache.nixos.org http://i.q8.fyi:5000
trusted-public-keys = cache.nixos.org-1:... i.q8.fyi:PUBLIC_KEY
```

Previous versions remain in `/nix/store`. To roll back:

```bash

gcloud compute ssh $VM_NAME --zone=$GCP_ZONE -- "ls -la /nix/store/*iroh-relay*"

# Activate it
gcloud compute ssh $VM_NAME --zone=$GCP_ZONE -- "sudo cp /nix/store/<previous-path>/bin/iroh-relay /usr/local/bin/ && sudo systemctl restart iroh-relay"
```
